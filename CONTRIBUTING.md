# Contributing

## Principles

- Keep changes focused and reviewable.
- Update docs when paths, behavior, or APIs change.
- Include validation commands in PR descriptions.

## Development Workflow

1. Open or link an issue.
2. Implement in small increments.
3. Run tests/docs build.
4. Submit PR with rationale and test evidence.

## Docs + Image Policy

Docs images should follow Tau-Ceti visual style. See `docs/image-style.md`.
