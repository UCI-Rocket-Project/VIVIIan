# Contributing

## Workflow

1. Open an issue describing user impact.
2. Submit small, focused PRs.
3. Include tests and docs updates for behavior changes.

## Quality Bar

- Clear naming and module boundaries.
- No hidden side effects.
- Reproducible validation steps in PR description.

## Docs Requirements

- Update docs when paths, APIs, or workflows change.
- Keep screenshots and diagrams readable with high contrast and concise labels.
